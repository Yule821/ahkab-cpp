## 1. 什么是电路仿真？

### 1.1 直观理解

假设你要设计一个电路——比如一个音频放大器。你有两种选择：

- **实物搭建**：买元件、焊电路板、上电测试。烧了再买。
- **软件仿真**：在电脑里画出电路，让程序帮你算电压电流。改参数不花钱。

**电路仿真器**就是做后者的软件。你告诉它"这里有个 5V 电池，那里有个 1kΩ 电阻"，它就能告诉你每个点的电压是多少。

### 1.2 SPICE 是什么

[SPICE](https://en.wikipedia.org/wiki/SPICE)（Simulation Program with Integrated Circuit Emphasis）诞生于 1970 年代加州大学伯克利分校，是电路仿真的事实标准。今天几乎所有的电路仿真软件（LTspice、HSPICE、ngspice）都兼容 SPICE 格式。

**ahkab-cpp** 就是一个 SPICE 风格的仿真器，用 C++ 写成。它包含了 SPICE 最核心的功能——直流分析、交流分析、瞬态分析、零极点分析，可以处理电阻、电容、电感、二极管、MOSFET 和 BJT。

### 1.3 你需要会什么

- 会打开终端 / 命令行
- 了解最基本的电学概念：电压（V）、电流（A）、电阻（Ω）、电容（F）、电感（H）
- 能区分直流和交流

如果你连电压电流都不清楚，建议先花 10 分钟看一个电路基础科普视频再回来——会轻松很多。

## 3. SPICE 网表语法入门

### 3.1 什么是网表？

**网表（Netlist）**是一个纯文本文件，用来描述电路长什么样。你不用画图——写几行文字就够了。

### 3.2 网表基本结构

```
* 这是注释（星号开头）

第一行永远是标题
R1  1  0  1k          ; 在节点1和0之间放一个1kΩ电阻
V1  1  0  dc 5        ; 在节点1和0之间放一个5V直流电压源

.OP                    ; 执行直流工作点分析
.END                   ; 网表结束
```

### 3.3 节点编号规则

电路中的每个连接点叫一个**节点**（Node），用名字或数字标识：

- 节点 `0`（或 `GND`）永远是**地**（0V 参考点）——每个电路必须有一个接地点
- 其他节点可以任意命名：`1`、`in`、`output`、`n001` 都行
- 每个元件必须用两个（或更多）节点连接起来

```
R1  A  B  100
```
意思是"在节点 A 和节点 B 之间放一个 100Ω 的电阻"。

### 3.4 数值后缀

SPICE 支持工程后缀（不区分大小写）：

| 后缀 | 含义 | 示例 |
|------|------|------|
| T | 10^12（太） | `1T` = 1000000000000 |
| G | 10^9（吉） | `1G` = 1000000000 |
| MEG | 10^6（兆） | `1MEG` = 1000000 |
| K | 10^3（千） | `1K` = 1000 |
| M | 10^-3（毫） | `1M` = 0.001 |
| U | 10^-6（微） | `1U` = 0.000001 |
| N | 10^-9（纳） | `1N` = 0.000000001 |
| P | 10^-12（皮） | `1P` = 0.000000000001 |
| F | 10^-15（飞） | `1F` = 0.000000000000001 |

例子：`1K` = 1000Ω，`10U` = 0.00001F，`2.2MEG` = 2200000。

### 3.5 分析命令

| 命令 | 含义 | 用法 |
|------|------|------|
| `.OP` | 直流工作点 | 计算电路中各节点直流电压 |
| `.DC` | 直流扫描 | 改变某个源的电压，观察输出变化 |
| `.TRAN` | 瞬态分析 | 观察电压/电流随时间的变化 |
| `.AC` | 交流分析 | 观察电路对不同频率信号的响应 |
| `.PZ` | 零极点分析 | 分析传递函数的零点和极点 |
| `.END` | 结束标记 | 网表末尾必须有这一行 |

## 4. 第一个仿真：直流工作点 (OP)

### 4.1 什么是工作点？

**工作点**（Operating Point）是电路在直流稳态下的电压电流分布。相当于你不给它任何变化的信号，只看静止状态下每个节点的电压。

### 4.2 示例：电阻分压器

创建文件 `op_example.cir`：

```
* 电阻分压器工作点分析
V1  1  0  dc 9         ; 9V 电池，正极接节点1，负极接地
R1  1  2  1k            ; 1kΩ 电阻，一端接节点1，另一端接节点2
R2  2  0  2k            ; 2kΩ 电阻，一端接节点2，另一端接地

.OP
.END
```

**电路解释**：

```
    9V
  V1 +----
    |    |
    |   R1 (1k)
    |    |
    |   [2] ---- 这就是我们要算的电压点
    |    |
    |   R2 (2k)
    |    |
  V1 ----- 地 (0V)
```

根据分压公式：`V(2) = 9 × 2000 / (1000 + 2000) = 6V`

### 4.3 运行仿真

```bash
cd build
./src/cli/ahkab_cli ../op_example.cir
```

输出：
```
OP Results:
Node   Voltage(V)
0      0.000000e+00
1      9.000000e+00
2      6.000000e+00
```

节点 2 确实是 6V，与理论一致。

## 5. 直流扫描 (DC Sweep)

### 5.1 什么是直流扫描？

直流扫描就是**固定其他条件，让一个电压源（或电流源）从 A 慢慢变到 B，观察某个节点的电压怎么变**。

通俗比方：你想知道把音量旋钮从 0 拧到最大时，喇叭两端的电压变化曲线。直流扫描就是反复在每一档"音量"下做一个工作点分析，然后画成图。

### 5.2 示例：分压器直流扫描

创建 `dc_example.cir`：

```
* 直流扫描示例
V1  1  0  dc 0        ; 电压源从0V开始（扫描时会覆盖它）
R1  1  2  1k
R2  2  0  2k

.DC V1 0 5 1           ; 扫描V1：从0V到5V，每步1V
.END
```

`.DC V1 0 5 1` 的含义：
- `V1`：扫描的目标源
- `0`：起始电压
- `5`：终止电压
- `1`：步长

```bash
./src/cli/ahkab_cli ../dc_example.cir
```

输出：
```
DC Sweep Results:
V-sweep     Node 1      Node 2
0.000e+00   0.000e+00   0.000e+00
1.000e+00   1.000e+00   6.667e-01
2.000e+00   2.000e+00   1.333e+00
3.000e+00   3.000e+00   2.000e+00
4.000e+00   4.000e+00   2.667e+00
5.000e+00   5.000e+00   3.333e+00
```

可以看到：`V(2) = V(1) × 2/3` 始终成立，这正是分压器的特性。

## 6. 瞬态分析 (TRAN)

### 6.1 什么是瞬态分析？

**瞬态分析**计算电路电压/电流随时间的变化。比如：
- 电源刚接通时，电容两端的电压怎么上升？
- 一个振荡器输出什么样的波形？
- 一个开关电路从开到关的过渡过程长什么样？

### 6.2 示例：RC 充电电路

创建 `tran_example.cir`：

```
* RC 充电瞬态分析
V1  1  0  dc 5          ; 5V 直流电源
R1  1  2  1k            ; 1kΩ 电阻
C1  2  0  100u          ; 100μF 电容

.TRAN 1m 0.2 0           ; 步长1ms，仿真到0.2秒
.END
```

`.TRAN 1m 0.2 0` 含义：
- `1m`（1ms）：时间步长（每一步模拟 1 毫秒）
- `0.2`（0.2s）：仿真总时长
- `0`：起始时间（0 秒）

**物理原理**：RC 充电公式：
```
Vc(t) = Vs × (1 - e^(-t/RC))
```
其中 `RC = 1kΩ × 100μF = 0.1s`（时间常数）。

- 经过 1 个时间常数 (0.1s)：电压达到约 63.2% 的最终值
- 经过 5 个时间常数 (0.5s)：电压达到约 99.3% 的最终值

```bash
./src/cli/ahkab_cli ../tran_example.cir
```

输出（部分）：
```
Time        V(1)        V(2)        I(V1)
0.000e+00   5.000e+00   0.000e+00   ...
1.000e-02   5.000e+00   4.750e-01   ...
...
1.000e-01   5.000e+00   3.160e+00   ...
2.000e-01   5.000e+00   4.320e+00   ...
```

可以看到 V(2)（电容电压）从 0V 开始，逐渐逼近 5V。在 t=0.1s 时约 3.16V，符合 63.2% 的预期。

### 6.3 将输出保存为 CSV

```bash
./src/cli/ahkab_cli -o output.csv ../tran_example.cir
```

然后用 Excel 或任何工具打开 `output.csv` 画图。

## 7. 交流频率分析 (AC)

### 7.1 什么是交流分析？

**交流分析**（AC Analysis）回答"电路对不同频率信号的放大/衰减程度"：

- 比如音频放大器——它对 100Hz 低音和 10kHz 高音的放大倍数一样吗？
- 比如低通滤波器——它允许低频通过、阻挡高频，具体在哪个频率开始衰减？

AC 分析输出的是**复数**（幅度和相位），所以结果通常同时显示增益（dB）和相位（度）。

### 7.2 示例：RC 低通滤波器

创建 `ac_example.cir`：

```
* RC 低通滤波器 AC 分析
V1  1  0  dc 0   ac 1  0    ; DC=0，AC幅度=1V，AC相位=0
R1  1  2  1k                ; 1kΩ 电阻
C1  2  0  1u                ; 1μF 电容

.AC DEC 10 1 1MEG           ; 十倍频10点，从1Hz到1MHz
.END
```

`.AC DEC 10 1 1MEG` 含义：
- `DEC`：对数间隔（每十倍频取点）
- `10`：每十倍频取 10 个点
- `1`：起始频率 1Hz
- `1MEG`：终止频率 1MHz

**物理原理**：RC 低通的截止频率 `fc = 1 / (2πRC) = 1 / (2π × 1000 × 10^-6) ≈ 159Hz`。低于 159Hz 的信号几乎无损通过，高于 159Hz 的信号每十倍频衰减 20dB。

```bash
./src/cli/ahkab_cli ../ac_example.cir
```

输出（部分）：
```
AC Analysis Results:
Frequency   |V(2)|      Phase(V2)
1.000e+00   9.999e-01   -3.598e-02
...
1.585e+02   7.071e-01   -4.500e+01   ; 截止频率处，增益=0.707 (-3dB)
...
1.000e+04   1.587e-02   -8.910e+01
```

在截止频率 159Hz 处，输出幅度约 0.707（-3dB），相位约 -45 度，完全符合理论。

## 8. 半导体器件仿真

### 8.1 二极管 (D)

创建 `diode_example.cir`：

```
* 二极管半波整流
V1  1  0  dc 0
D1  1  2  DMOD             ; 二极管，正极接节点1，负极接节点2
R1  2  0  1k               ; 1kΩ 负载电阻
.MODEL DMOD D (IS=1e-14)   ; 二极管模型定义

.OP
.END
```

`.MODEL` 行定义了一个名为 `DMOD` 的二极管理想模型，并设定了饱和电流 IS=10^-14 A。

> 二极管 SPICE 参数说明：
> - `IS`：饱和电流（默认 10^-14 A）
> - `N`：发射系数（默认 1）
> - `RS`：串联电阻
> - `CJO`：零偏结电容
> - `VJ`：结电势
> - `BV`：反向击穿电压
> - `IBV`：击穿时的电流

### 8.2 MOSFET (M)

```
* NMOS 放大器偏置
VDD  1  0  5
M1   2  3  0  0  NMODEL W=10U L=1U   ; NMOS: 漏=D(2), 栅=G(3), 源=S(0), 衬底=B(0)
.MODEL NMODEL NMOS (VTO=0.7 KP=50U)
```

MOSFET 语法：`M<id> drain gate source bulk model [W=<w> L=<l>]`

### 8.3 BJT (Q)

```
* NPN 共射放大器
VCC  1  0  10
Q1   2  3  0  NPNMODEL   ; NPN: 集电极=C(2), 基极=B(3), 发射极=E(0)
.MODEL NPNMODEL NPN (BF=100 IS=1e-14)
```

BJT 语法：`Q<id> collector base emitter [substrate] model`

## 9. 零极点分析 (PZ)

### 9.1 什么是零极点分析？

对于线性电路，其频率特性可以用**传递函数** H(s) 来描述。传递函数的根分为：

- **零点（Zeros）**：使 H(s)=0 的频率点
- **极点（Poles）**：使 H(s)=∞ 的频率点

极点决定了电路的稳定性和带宽。如果某个极点落在右半平面（实部为正），电路就会振荡（不稳定）。

### 9.2 示例

```
* RC 电路零极点分析
V1  1  0  dc 0  ac 1
R1  1  2  1k
C1  2  0  1u

.PZ V(2) V1              ; 以V1为输入，V(2)为输出
.END
```

输出会给出传递函数的极点和零点位置。

## 11. 常见问题排查

### 11.1 "收敛失败" (Convergence Error)

```
Error: Convergence failed at step 500, residual = 1.234e-05
```

**原因**：Newton-Raphson 迭代无法在限定次数内找到解。

**解决方法**：
1. 检查电路是否正确连接——每个节点都至少连接了两个元件
2. 确保有接地节点（节点 `0`）
3. 给非线性器件添加串联电阻（如给二极管串一个 1Ω）
4. 观察电路是否有"浮空"节点——没有任何 DC 路径到地
5. 电容两端并联一个大电阻（如 1GΩ）提供 DC 通路

### 11.2 "奇异矩阵" (Singular Matrix)

**原因**：电路中存在电压源回路或电流源割集，导致 MNA 矩阵不可逆。

**解决方法**：
1. 不要把两个电压源直接并联
2. 不要在同一个回路上串联多个电感——给电感串一个小电阻
3. 确保每个节点都有 DC 路径到地

### 11.3 "0 节点不存在"

**原因**：没有定义接地节点。

**解决方法**：确保电路中有一个元件的一脚连到节点 `0`，或者显式定义 `ground = 0`。

### 11.4 仿真结果为 0

**原因**：可能没有交流激励。

**解决方法**：AC 分析时，至少给一个独立源指定 `ac` 参数，如：
```
V1 1 0 dc 5 ac 1 0    ; DC=5V, AC=1V 幅度, 相位=0
```

### 11.5 数值积分方法选择

在 `.OPTIONS` 中可以设定：
```
.OPTIONS method=TRAP       ; 梯形法（默认，精度和速度平衡）
.OPTIONS method=GEAR       ; Gear 法（刚性问题更稳定）
```

| 方法 | 优点 | 缺点 |
|------|------|------|
| TRAP（梯形） | 精度高，A 稳定 | 可能产生虚假振荡 |
| GEAR | 处理刚性电路好 | 低阶时精度下降 |
| Euler Backward | 简单 | 精度最差 |
